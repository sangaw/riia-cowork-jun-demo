Defect Analysis — Market Analysis Instrument Selector

  Root Cause 1 (Primary): selectInstrumentTab() never reloads market signals

  main.js:84-93 — when you click an instrument tab, the function:
  - Saves to localStorage ?
  - Updates tab CSS ?
  - Calls health/drift/progress loaders ?
  - Never calls loadMarketSignals() ?

  The section loader (_sectionLoaders['market-signals']) fires only once on first visit (nav.js pattern). After that,
  switching instruments does nothing to the charts — they keep showing whichever instrument was active on first visit.

  Why NVIDIA might "work": If NVIDIA was still in localStorage from a previous session, the page loads with NVIDIA as
  active, market-signals fires once and fetches NVIDIA data.

  ---
  Root Cause 2 (Secondary): No CSV for NVIDIA in data/raw/

  find_instrument_csv('NVIDIA') will find data/raw/NVIDIA/ directory (it exists), but no CSV inside it — the raw dir
  listing shows ASML/, BANKNIFTY/, NIFTY/, NVIDIA/ but only ASML, BANKNIFTY, NIFTY have CSV files. It also checks
  data/input/NVIDIA/ which doesn't exist ? FileNotFoundError ? API returns [] ? all KPIs show —.

  ---
  Fix Plan

  Fix 1 — main.js (1 line change): Add await loadMarketSignals() at the end of selectInstrumentTab().

  Fix 2 — observability.py fallback for BANKNIFTY: The data/input/DAILY-DATA/banknifty_manual.csv exists but
  find_instrument_csv('BANKNIFTY') looks in data/raw/BANKNIFTY/ first — that has banknifty_daily_25yr_rounded.csv, so
  BANKNIFTY should work once Fix 1 is in.

  Fix 3 — NVIDIA has no CSV data: Either add a CSV to data/raw/NVIDIA/ or remove NVIDIA from the instrument tabs in
  rita.html:397. The API returns [] and the JS silently exits at if (!rows || !rows.length) return — no error shown.

  Fix 4 (nice-to-have): When loadMarketSignals() returns empty data, show a "No data available for [INSTRUMENT]" message
   instead of silently leaving stale data on screen.
