# RITA — Data Specification

High-density reference for AI agents. Read before any code that reads, writes, or seeds data files.

---

## 1. Directory Layout

```
riia-jun-release/
└── data/
    ├── raw/          ← SOURCE DATA — read-only, never modify or delete
    │   ├── NIFTY/
    │   ├── BANKNIFTY/
    │   ├── ASML/
    │   └── NVIDIA/
    ├── input/        ← PROCESSED / DAILY-UPDATED — read-only for the API
    │   ├── DAILY-DATA/   ← daily FnO records + recent manual OHLCV
    │   ├── NIFTY/        ← empty (.gitkeep) — future processed output
    │   ├── BANKNIFTY/    ← empty (.gitkeep)
    │   ├── ASML/
    │   └── NVIDIA/
    └── output/       ← WRITTEN BY THE API — model artefacts, backtest results
        ├── NIFTY/
        └── BANKNIFTY/
```

**Rules:**
- `data/raw/` — never write to this directory from code. It is the ground-truth source.
- `data/input/` — read-only from the API's perspective. Updated manually by the user (e.g., appending new daily data to `nifty_manual.csv`).
- `data/output/` — the only directory the API writes to (model `.zip` files, backtest artefacts).

## 4. `load_nifty_csv()` — The Canonical Data Loader

**Location:** `src/rita/core/data_loader.py`

All code that reads OHLCV CSVs must use this function. Do not write ad-hoc `pd.read_csv()` calls in routers or services.

```python
from rita.core.data_loader import load_nifty_csv

df = load_nifty_csv("data/raw/NIFTY/merged.csv")
# Returns: DataFrame with DatetimeIndex, columns = Open, High, Low, Close, Volume
# Index is timezone-naive (converted from IST if needed)
# Sorted ascending by date
```

**What it handles automatically:**
- Lowercase → Title Case column renaming (`close` → `Close`, `shares traded` → `Volume`)
- Timezone-aware IST dates (`+05:30`) → tz-naive
- Plain ISO dates (`2001-03-09`) → datetime
- `dd-MMM-yyyy` format (`16-MAR-2026`) → datetime
- Sorts by date ascending

**Output columns:** `Open`, `High`, `Low`, `Close`, `Volume` (Volume may be NaN if the source file has no volume column)

## 5. DB Seeding — What Goes In and Why

Only NIFTY data is seeded into the DB at startup. Other instruments are not seeded because only NIFTY is the active trading instrument.

**Seeding window: 2025 + 2026 only**

| Source | Year filter | Rows | Reason |
|---|---|---|---|
| `data/raw/NIFTY/merged.csv` | year == 2025 | 249 | Full year of recent history |
| `data/input/DAILY-DATA/nifty_manual.csv` | year == 2026 | ~17 | Current year data |
| **Total** | | **~266** | |

**Why not the full 25-year history?**
Seeding 6,594 rows row-by-row (one `upsert()` = one `commit()`) took 78 seconds at startup. The new bulk insert is < 2 seconds. 266 rows is sufficient for all technical indicators:

| Indicator | Bars needed | Available |
|---|---|---|
| RSI-14 | 14 | 266 |
| MACD (12, 26, 9) | 26 | 266 |
| Bollinger Bands (20, 2σ) | 20 | 266 |
| EMA-50 | 50 | 266 |
| ATR-14 | 14 | 266 |

**Adding new year data:** When 2026 ends, update the seed filter in `main.py` to `year >= 2025` → `year >= 2026`, and move 2026 data from `nifty_manual.csv` into `merged.csv` (or create a new `merged_2026.csv`).

---

## 6. Adding New Daily Data (Ongoing Process)

When a new trading day closes, append the OHLCV row to `nifty_manual.csv` manually. Format must match the existing file (`dd-MMM-yyyy` date, same column order). The DB does not auto-update — it is seeded once on startup. The `market_signals` API reads from the DB; new rows become visible only after a DB reset + restart or a direct DB insert.

---

## 7. Model Output Files

Trained model artefacts are written to `data/output/{INSTRUMENT}/`:

```
data/output/NIFTY/
    {model_version}.zip       ← stable-baselines3 DoubleDQN model
    backtest_results.csv      ← daily portfolio values from latest backtest
    performance_summary.json  ← aggregated KPIs
```

**These files survive DB deletion.** The DB tracks run metadata (timestamps, Sharpe, MDD) but the model weights and backtest CSVs are on disk. If the DB is lost, the models can still be loaded and used — only the run history is lost.

## 8. AI Agent Directives

1. **Never write to `data/raw/`** — it is the immutable source of truth.
2. **Never call `pd.read_csv()` directly on OHLCV files** — always use `load_nifty_csv()`. It handles all date formats and column normalisation.
3. **Do not seed the full 25-year history** — the 2025+2026 window is the correct design. Full history causes slow startup.
4. **To add a new instrument to the DB seed:** follow the same pattern in `main.py` lifespan — check if table empty, bulk `db.add_all()`, single `db.commit()`.
5. **`nifty_manual.csv` is the live feed** — it is the only file updated after each trading day. Code that needs the most recent data should read from the DB (which includes it) not directly from the file.
6. **Check `data/raw/` before downloading** — all instruments already have 25 years of historical data. Never suggest downloading from NSE or Yahoo Finance unless explicitly asked.
